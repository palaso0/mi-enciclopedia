RLHF: Aprendizaje por refuerzo con retroalimentación humana

Un modelo de lenguaje pre-entrenado puede generar texto fluido, pero no sabe si lo que genera es útil, seguro o apropiado. Es como un niño muy leído que ha memorizado millones de libros pero nunca ha recibido una retroalimentación sobre qué comportamientos son aceptables. RLHF (Reinforcement Learning from Human Feedback) es el proceso que enseña al modelo a generar respuestas que los humanos realmente valoran.

1. La analogía: Niño con calcomanías

Analogía: Imagina un niño que sabe hablar fluidamente pero dice cosas sin filtro. Para enseñarle buen comportamiento, le das calcomanías doradas cuando dice algo amable y quitas calcomanías cuando dice algo grosero. Con el tiempo, el niño aprende a generar automáticamente frases que le ganan más calcomanías. Eso es RLHF: el modelo aprende a generar respuestas que maximizan la "recompensa" humana.

2. Los tres pasos del RLHF

RLHF no es un solo proceso; son tres etapas secuenciales, cada una con un objetivo distinto:

Paso 1 de 3
SFT — Supervised Fine-Tuning
Entrenadores humanos crean un dataset de ejemplos de alta calidad (pregunta → respuesta ideal). El modelo pre-entrenado se ajusta con estos ejemplos para aprender el formato y el estilo de una conversación útil.
↓
Paso 2 de 3
Modelo de Recompensa (Reward Model)
Humanos evalúan múltiples respuestas del modelo para la misma pregunta, ranqueándolas de mejor a peor. Con estos datos se entrena un modelo separado que aprende a predecir qué respuesta preferiría un humano. Este modelo de recompensa asigna un puntaje (reward) a cada respuesta.
↓
Paso 3 de 3
PPO — Proximal Policy Optimization
El modelo generador produce respuestas, el modelo de recompensa las puntúa, y el algoritmo PPO ajusta los parámetros del generador para maximizar la recompensa. El truco es que PPO limita cuánto puede cambiar el modelo en cada paso, evitando que "haga trampas" para obtener puntos altos sin realmente mejorar.

3. El modelo de recompensa: Cómo aprende a puntuar

El corazón del RLHF es el modelo de recompensa. Este modelo recibe una pregunta y una respuesta, y genera un número que representa cuán "buena" es esa respuesta según los estándares humanos.

Problema del reward hacking: Si el modelo de recompensa no es robusto, el generador puede aprender a "engañarlo". Por ejemplo, generar respuestas muy largas y verbosas que el reward model confunda con respuestas completas y útiles. Este fenómeno se llama reward hacking y es uno de los mayores desafíos del RLHF.

4. PPO: Aprender sin descontrolarse

El algoritmo PPO (Proximal Policy Optimization) es el que realmente ajusta los pesos del modelo generador. La idea clave es: mejora poco a poco. En cada paso, PPO calcula cuánto cambió el comportamiento del modelo y penaliza los cambios demasiado grandes.

Esto es importante porque sin esta restricción, el modelo podría encontrar "atajos" para obtener recompensas altas sin realmente aprender a generar mejores respuestas. PPO asegura que cada actualización sea un paso moderado en la dirección correcta.

  • Ratio de probabilidad: Mide cuánto cambió la probabilidad de generar una respuesta después del ajuste.
  • Clip (recorte): Si el ratio se desvía demasiado de 1.0, PPO lo recorta, impidiendo cambios drásticos.
  • Coeficiente KL: Una penalización que mantiene al modelo cerca de su comportamiento original, evitando que se desvíe demasiado del SFT.

5. Constitutional AI: RLHF sin humanos

RLHF tiene un problema de escala: necesita humanos evaluando respuestas continuamente, lo cual es caro y lento. Constitutional AI (CAI), propuesto por Anthropic, reemplaza parte de la retroalimentación humana con un conjunto de principios escritos (una "constitución") que el propio modelo usa para auto-evaluarse.

Ejemplo de constitución:

  • "Elige la respuesta que sea más útil, honesta e inofensiva."
  • "Evita generar contenido que promueva odio o violencia."
  • "Si no sabes la respuesta, di que no la sabes en vez de inventar."
  • "Respeta la privacidad y no reveles información personal."

En CAI, el modelo genera respuestas, luego se auto-critica usando la constitución, y luego genera una versión mejorada. Este ciclo de auto-mejora reduce drásticamente la necesidad de evaluadores humanos, manteniendo resultados de alta calidad.

# Simulación simplificada del pipeline de RLHF
# Paso 1: SFT (Supervised Fine-Tuning) - ya completado
# Paso 2: Entrenar modelo de recompensa
# Paso 3: Optimizar con PPO

class RewardModel:
    def __init__(self):
        # En la práctica, esto es una red neuronal
        # Aquí simulamos con una función de puntuación
        self.peso_utilidad = 0.4
        self.peso_seguridad = 0.35
        self.peso_honestidad = 0.25

    def puntuar(self, respuesta, es_util, es_segura, es_honesta):
        score = (
            self.peso_utilidad * es_util +
            self.peso_seguridad * es_segura +
            self.peso_honestidad * es_honesta
        )
        return round(score, 2)

# Simular 3 respuestas a la misma pregunta
rm = RewardModel()

respuestas = [
    {"texto": "La Tierra gira alrededor del Sol en ~365 días",
     "util": 0.95, "segura": 1.0, "honesta": 0.98},
    {"texto": "Creo que es al revés, el Sol gira alrededor de la Tierra",
     "util": 0.1, "segura": 1.0, "honesta": 0.05},
    {"texto": "No estoy seguro, pero déjame inventar algo",
     "util": 0.3, "segura": 0.8, "honesta": 0.1}
]

print("Puntuaciones del Modelo de Recompensa:")
print("-" * 50)
for i, r in enumerate(respuestas, 1):
    score = rm.puntuar(r["texto"], r["util"], r["segura"], r["honesta"])
    print(f"Respuesta {i}: {score:+.2f}")
    print(f"  → \"{r['texto'][:50]}...\"")

# El modelo de recompensa guía al generador:
# respuestas precisas y honestas reciben recompensa alta,
# respuestas incorrectas o deshonestas reciben penalización.

Impacto: RLHF fue lo que transformó a GPT-3 en ChatGPT. El modelo base podía generar texto, pero RLHF lo hizo seguro, útil y conversacional. Sin este paso, los modelos generan contenido ofensivo, inventan datos, o simplemente no entienden qué se espera de ellos en una interacción real.
