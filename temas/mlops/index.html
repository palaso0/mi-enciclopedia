MLOps: Machine Learning Operations

En resumen: MLOps es la práctica de llevar un modelo de ML desde el laboratorio hasta producción de forma confiable, reproducible y monitoreada. Es DevOps aplicado a machine learning.

1. El problema

Un数据科学家entrena un modelo en su laptop, obtiene buenas métricas, y lo comparte con el equipo de ingeniería. Ahora empiezan los problemas:

  • El modelo funciona en el notebook pero no en producción
  • Nadie sabe qué datos se usaron para entrenar
  • El modelo se degrada con el tiempo pero nadie lo detecta
  • Reentrenar toma días porque no hay pipeline automatizado
  • No hay forma de volver a una versión anterior si algo sale mal

MLOps resuelve estos problemas con automatización, versionado y monitoreo.

3. Versionado de datos y modelos

Data Versioning (DVC)

Así como Git versiona código, DVC versiona datos y modelos. Cada experimento queda registrado con qué datos, qué código y qué resultados produjo.


# Inicializar DVC en tu proyecto
dvc init

# Agregar un archivo de datos grande
dvc add data/entrenamiento.csv

# El archivo real va a un storage remoto (S3, GCS, etc.)
# Git solo guarda el puntero (.dvc)
git add data/entrenamiento.csv.dvc
git commit -m "Agregar datos de entrenamiento v1"

Experiment Tracking

Cada experimento queda registrado: qué hiperparámetros se usaron, qué métricas se obtuvieron, qué modelo se generó.


import mlflow

mlflow.set_experiment("clasificacion-spam")

with mlflow.start_run():
    mlflow.log_param("modelo", "random_forest")
    mlflow.log_param("n_estimators", 100)

# FastAPI para servir un modelo
from fastapi import FastAPI
import joblib

app = FastAPI()
modelo = joblib.load("modelo.pkl")

@app.post("/predecir")
def predecir(datos: dict):
    resultado = modelo.predict([datos["features"]])
    return {"prediccion": resultado.tolist()}

5. Monitoreo y drift

Model Drift

Un modelo que funcionaba bien ayer puede empezar a funcionar mal hoy. Los datos cambian, el comportamiento de los usuarios cambia, el mundo cambia. El monitoreo detecta cuándo el modelo ya no es confiable.

  • Data drift: La distribución de los datos de producción cambió respecto a los de entrenamiento.
  • Concept drift: La relación entre variables y target cambió.
  • Performance drift: Las métricas del modelo bajan (accuracy, F1, etc.).

# Ejemplo de monitoreo de drift con Evidently
from evidently.report import Report
from evidently.metric_preset import DataDriftPreset

report = Report(metrics=[DataDriftPreset()])
report.run(
    reference_data=datos_entrenamiento,
    current_data=datos_produccion
)
report.save_html("drift_report.html")

7. Herramientas del ecosistema

  • MLflow: Experiment tracking, model registry, deployment.
  • DVC: Versionado de datos y pipelines de ML.
  • Kubeflow: Pipelines de ML en Kubernetes.
  • Weights & Biases: Tracking de experimentos y visualización.
  • Evidently / Great Expectations: Validación de datos y monitoreo de drift.
  • Feast: Feature store para servir features en producción.

La meta de MLOps: Poder reproducir cualquier experimento, deployear un modelo nuevo en minutos (no semanas), y saber inmediatamente cuándo un modelo deja de funcionar.
