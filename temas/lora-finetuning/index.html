LoRA: Fine-tuning eficiente

Entrenar un modelo de lenguaje desde cero cuesta millones de dólares. Pero una vez que tienes un modelo base potente, ¿y si solo necesitas adaptarlo a una tarea específica? El fine-tuning tradicional requiere actualizar todos los parámetros del modelo, lo cual sigue siendo extremadamente costoso. LoRA (Low-Rank Adaptation) resuelve esto con una idea elegante: no reconstruyas el motor, agrega un plugin.

1. El problema: Fine-tuning completo es desperdiciar recursos

Cuando haces fine-tuning completo (full fine-tuning) de un modelo como Llama 3 de 70B parámetros, necesitas:

Fine-tuning Completo
Parámetros a entrenar 70,000M (100%)
VRAM necesaria ~280 GB+
Tiempo estimado Días/semanas
Riesgo de olvido Alto
Fine-tuning con LoRA
Parámetros a entrenar ~200M (0.3%)
VRAM necesaria ~24 GB
Tiempo estimado Horas
Riesgo de olvido Bajo

El truco de LoRA: En lugar de actualizar los 70B parámetros, LoRA congela todos los pesos originales del modelo y entrena únicamente unas matrices pequeñas de rango bajo que se suman a las capas originales. El modelo base no cambia; solo se le añade un "adaptador" ligero.

2. La analogía: Plugin vs. reconstrucción

Analogía: Imagina que tienes un auto de alta gama (el modelo base) y quieres que también pueda conducir por terreno off-road. La opciones son: (A) desarmar el motor y reconstruirlo para off-road, o (B) instalar un kit de elevación y neumáticos todoterreno (el adaptador LoRA). La opción B es más barata, reversible, y no destruye las capacidades originales del auto. Si quieres volver al asfalto, solo quitas el kit.

En términos técnicos, LoRA observa que los cambios necesarios para adaptar un modelo a una nueva tarea tienen una baja dimensión intrínseca. No necesitas cambiar todos los parámetros; solo necesitas ajustar unas cuantas direcciones en el espacio de pesos.

3. Cómo funciona: Matrices de bajo rango

En una capa Transformer, los pesos de atención son matrices grandes (por ejemplo, 4096 x 4096). LoRA no modifica estas matrices. En su lugar, introduce dos matrices pequeñas, A y B, y calcula la adaptación como una suma:

W' = W + B × A

Con r=8: Parámetros entrenados = 4096×8 + 8×4096 = 65,536 vs. 4096×4096 = 16,777,212 originales

Las matrices A y B son muy pequeñas comparadas con W. Mientras W tiene millones de parámetros, A y B tienen solo miles. El rango r (típicamente 8, 16 o 64) controla cuánta capacidad de adaptación tiene el adaptador.

4. Implementación práctica con PEFT

La biblioteca PEFT (Parameter-Efficient Fine-Tuning) de Hugging Face hace que aplicar LoRA sea casi trivial. Solo necesitas definir qué capas adaptar y el rango deseado:

# Ejemplo de fine-tuning con LoRA usando PEFT + transformers
from peft import LoraConfig, get_peft_model, TaskType
from transformers import AutoModelForCausalLM

# 1. Cargar el modelo base (congelado)
modelo_base = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-3-8B",
    load_in_4bit=True  # Carga quantizado para ahorrar VRAM
)

# 2. Configurar LoRA: qué capas adaptar y con qué rango
config_lora = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=16,                          # Rango de las matrices A y B
    lora_alpha=32,                 # Factor de escala (alpha/r = 2)
    lora_dropout=0.05,             # Regularización
    target_modules=[               # Capas del Transformer a adaptar
        "q_proj", "k_proj",        # Proyecciones de atención
        "v_proj", "o_proj",
        "gate_proj", "up_proj",    # Capas FFN
        "down_proj"
    ],
    bias="none"
)

# 3. Aplicar LoRA al modelo base
modelo_lora = get_peft_model(modelo_base, config_lora)

# 4. Ver cuántos parámetros se entrenan vs. el total
modelo_lora.print_trainable_parameters()
# Salida: trainable params: 41,943,040 || all params: 8,071,319,552 || 0.52%

# 5. Entrenar normalmente con un dataset de tu tarea
# El modelo solo ajustará los ~42M parámetros de LoRA,
# manteniendo los 8B originales completamente intactos.

Ventaja clave: Como el modelo base no se modifica, puedes tener múltiples adaptadores LoRA para diferentes tareas (uno para español, otro para código, otro para resúmenes) e intercambiarlos instantáneamente sin recargar el modelo completo.

5. Cuándo usar LoRA vs. fine-tuning completo

LoRA no siempre es la mejor opción. Aquí está la guía de decisión:

  • Usa LoRA cuando: Tienes un modelo base potente y quieres adaptarlo a una tarea específica (clasificación, generación en un dominio, estilo particular). El dataset es pequeño o moderado.
  • Usa fine-tuning completo cuando: Necesitas cambiar fundamentalmente el comportamiento del modelo, o estás pre-entrenando desde un checkpoint muy pequeño.
  • Combina ambos: Muchos equipos hacen first pre-entrenamiento completo en datos genéricos, luego LoRA para adaptación específica.

LoRA también se puede combinar con quantización (QLoRA), lo que permite fine-tuning de modelos de 70B parámetros en una sola GPU de 24GB. Esta combinación democratizó el fine-tuning: antes era algo solo accesible para grandes empresas, ahora cualquier persona con una GPU decente puede adaptar modelos de última generación.
