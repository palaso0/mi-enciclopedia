Estructuras de Datos

Imagina que vas a un supermercado. Si el dependiente tiene todos los productos apilados en un solo monton, encontrar lo que necesitas sera un caos. Pero si todo esta organizado en estantes, pasillos y categorias, la busqueda se vuelve instantanea.

Las estructuras de datos son exactamente eso: formas de organizar la informacion dentro de una computadora para que pueda usarse de manera eficiente. No es solo guardar datos; es guardarlos de la forma correcta segun lo que vayas a hacer con ellos.

La regla de oro: No existe una estructura de datos perfecta para todo. Elegir la correcta depende de que operaciones necesitas hacer rapido: buscar, insertar, ordenar o recorrer. Cada estructura tiene sus fortalezas y debilidades.

1. El Array (Arreglo)

Un array es como una hilera de casillas numeradas. Cada casilla contiene un dato, y puedes acceder a cualquier casilla directamente si conoces su numero de posicion.

Array en memoria — frutas
indice ↓
0 manzana
1 platano
2 naranja
3 uva
4 cereza
↑ accedo directo con frutas[2]
📍
Analogia: Piensa en los apartamentos de un edificio. Si alguien te dice "departamento 5", vas directo ahi sin tener que recorrer todos los pisos. Acceso directo en tiempo constante O(1).

El problema: Si quieres insertar un elemento en medio, todo lo que esta despues se tiene que mover.

Insertar "fresa" en la posicion 1
0manzana
1fresa
2platano
3naranja
4uva
5cereza
↑ platano, naranja, uva y cereza se movieron una posicion → O(n) 

frutas = ["manzana", "platano", "naranja", "uva", "cereza"]

# Acceso directo por indice — O(1)
print(frutas[2])  # → "naranja"

# Insertar en medio — O(n), hay que mover elementos
frutas.insert(1, "fresa")
# → ["manzana", "fresa", "platano", "naranja", "uva", "cereza"]

2. Stack (Pila) y Cola (Queue)

Estas dos estructuras son las mas simples: solo puedes agregar y quitar elementos de un extremo.

Stack (Pila) — LIFO: Last In, First Out
push (meter) ↓
libro3 ← TOPE
libro2
libro1
↑ pop (sacar)
1. push("libro1") → [libro1]
2. push("libro2") → [libro1, libro2]
3. push("libro3") → [libro1, libro2, libro3]
4. pop() → libro3
Solo saca el ultimo
Queue (Cola) — FIFO: First In, First Out
← dequeue (salir) enqueue (entrar) →
ana
luis
pedro
maria
ana entro primero → sale primero | maria entro ultimo → sale ultimo 

# PILA — como una pila de platos
pila = []
pila.append("libro1")   # push
pila.append("libro2")
pila.append("libro3")
print(pila.pop())        # → "libro3" (el ultimo en entrar sale primero)

# COLA — como una fila de supermercado
from collections import deque
cola = deque()
cola.append("ana")        # enqueue
cola.append("luis")
cola.append("pedro")
print(cola.popleft())     # → "ana" (el primero en entrar sale primero)

Usos reales: Las pilas se usan para el "deshacer" (Ctrl+Z), evaluar expresiones matematicas y el call stack de funciones. Las colas se usan en colas de impresion, mensajes pendientes y sistemas de atencion al cliente.

class Nodo:
    def __init__(self, dato):
        self.dato = dato
        self.siguiente = None  # Puntero al siguiente nodo

# Creando: A → B → C
a = Nodo("A")
b = Nodo("B")

a.siguiente = b   # A apunta a B
b.siguiente = c   # B apunta a C

# Insertar X entre B y C
x = Nodo("X")
x.siguiente = c   # X apunta a C
b.siguiente = x   # B ahora apunta a X

# Resultado: A → B → X → C

4. Hash Table (Tabla Hash)

# Python: los diccionarios son tablas hash
usuarios = {}

usuarios["ana@mail.com"] = {"nombre": "Ana", "edad": 28}
usuarios["luis@mail.com"] = {"nombre": "Luis", "edad": 35}

# Busqueda instantanea — O(1)
print(usuarios["ana@mail.com"]["nombre"])  # → "Ana"

# Verificar si existe
if "pedro@mail.com" in usuarios:
    print("Encontrado")
else:
    print("No existe")  # ← Esto se ejecuta

Colisiones: A veces dos claves generan la misma posicion. Se resuelve con encadenamiento (una lista en ese slot) o direccion abierta (buscar el siguiente slot libre). En la practica, las colisiones son raras con una buena funcion hash.

5. Arbol (Tree)

Un arbol es una estructura jerarquica donde cada nodo puede tener multiples hijos. Es como un organigrama: una raiz en la cima, con ramas que se dividen hacia abajo.

Arbol binario de busqueda — Izquierda menor, derecha mayor

3

_
14
Buscar 6: 8 → 3 → 6 (3 pasos) | Buscar 14: 8 → 10 → 14 (3 pasos)
Comparacion: Buscar en lista vs. arbol
Lista lineal
1M de elementos
1,000,000
pasos maximo
Arbol balanceado
1M de elementos
~20
pasos maximo
Lista: O(n)    vs    Arbol: O(log n) 

class NodoArbol:
    def __init__(self, valor):
        self.valor = valor
        self.izquierdo = None   # Hijo izquierdo (menores)
        self.derecho = None     # Hijo derecho (mayores)

# Construir el arbol
raiz = NodoArbol(8)
raiz.izquierdo = NodoArbol(3)
raiz.derecho = NodoArbol(10)
raiz.izquierdo.izquierdo = NodoArbol(1)
raiz.izquierdo.derecho = NodoArbol(6)
raiz.derecho.derecho = NodoArbol(14)

# Buscar — cada paso descarta la mitad
def buscar(nodo, valor):
    if nodo is None: return False
    if valor == nodo.valor: return True
    if valor < nodo.valor:
        return buscar(nodo.izquierdo, valor)
    return buscar(nodo.derecho, valor)

print(buscar(raiz, 6))   # → True
print(buscar(raiz, 99))  # → False

6. Grafo (Graph)

Un grafo es la estructura mas flexible: nodos conectados por aristas. Puede representar redes sociales, mapas, dependencias de software, o cualquier relacion entre entidades.

Grafo — estaciones de metro
A
B
C
D
E
F
Camino mas corto de A a F: A → C → F | verde = visitado | rojo = actual 

# Grafo: diccionario de adyacencia
grafo = {
    "A": ["B", "C"],      # A conectado con B y C
    "B": ["A", "D", "E"], # B conectado con A, D, E
    "C": ["A", "F"],      # C conectado con A y F
    "D": ["B"],           # D solo con B
    "E": ["B", "F"],      # E con B y F
    "F": ["C", "E"]       # F con C y E
}

# BFS: encontrar camino mas corto
from collections import deque

def bfs(grafo, inicio, objetivo):
    visitados = set()
    cola = deque([[inicio]])
    visitados.add(inicio)
    while cola:
        camino = cola.popleft()
        nodo = camino[-1]
        if nodo == objetivo:
            return camino
        for vecino in grafo[nodo]:
            if vecino not in visitados:
                visitados.add(vecino)
                cola.append(camino + [vecino])
    return None

print(bfs(grafo, "A", "F"))  # → ['A', 'C', 'F']

7. Comparativa: Que elegir y cuando

Estructura Busqueda Insercion Cuando usarla
Array O(1) indice O(n) valor O(n) Datos fijos, acceso por posicion
Linked List O(n) O(1)* Insercion frecuente en medio
Hash Table O(1) O(1) Busqueda por clave, conteo, caches
Arbol (BST) O(log n) O(log n) Datos ordenados, busqueda eficiente
Grafo O(V+E) O(1) Relaciones, rutas, redes
Pila / Cola O(n) O(1) Undo, impresion, mensajes

8. Resumen

Acceso rapido por posicion

→ Usa un Array

Busqueda por clave o nombre

→ Usa una Hash Table

Datos jerarquicos o con orden

→ Usa un Arbol

Relaciones entre entidades

→ Usa un Grafo

Procesar en orden de llegada

→ Usa una Cola

Deshacer /回溯

→ Usa una Pila

Consejo practico: En la mayoria de lenguajes modernos, la estructura por defecto es una lista (array dinamico). Si tu programa es lento, antes de optimizar algoritmos, preguntate si estas usando la estructura correcta. Cambiar de array a hash table puede reducir el tiempo de horas a milisegundos.
